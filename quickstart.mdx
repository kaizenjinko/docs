Quickstart này hướng dẫn bạn hoàn tất đóng góp đầu tiên cho **Quant Docs** — từ lúc checkout repo đến khi xuất bản trang qua Pull Request.

## Điều kiện tiên quyết

Trước khi bắt đầu, bạn cần:

- Tài khoản GitHub được cấp quyền truy cập repo
- Git cài đặt trên máy local
- Trình soạn thảo Markdown hoặc IDE (VS Code, JetBrains, …)

Repo không yêu cầu runtime hay package. Toàn bộ nội dung là file MDX; Mintlify đảm nhận build và render.

## Quy trình đóng góp

<Steps>
  <Step title="Tạo nhánh">
    Checkout một nhánh mới từ `main`.

    ```bash
    git checkout -b docs/<chủ-đề>
    ```
  </Step>
  <Step title="Thêm trang">
    Tạo file `.mdx` ở thư mục gốc, đặt tên theo kebab-case và phản ánh chủ đề.

    ```bash
    touch risk-policy.mdx
    ```
  </Step>
  <Step title="Khai báo trong `docs.json`">
    Thêm đường dẫn vào `navigation.pages`. Gộp nhóm theo chủ đề thông qua `group` nếu cần.

    ```json
    {
      "group": "Risk & Compliance",
      "pages": ["risk-policy"]
    }
    ```
  </Step>
  <Step title="Review và merge">
    Mở Pull Request. Mintlify cung cấp bản preview tự động — kiểm tra hiển thị, yêu cầu review từ team lead trước khi merge vào `main`.
  </Step>
</Steps>

## Cấu trúc một trang

Mỗi file MDX gồm hai phần:

<CodeGroup>
```mdx Frontmatter
---
title: "Tiêu đề trang"
description: "Mô tả ngắn gọn"
---
```

```mdx Phần thân
Viết nội dung bằng Markdown, bổ sung component Mintlify khi cần.
```
</CodeGroup>

## Nguyên tắc biên tập

- Dùng giọng văn trang trọng, một ý một câu, ưu tiên danh sách liệt kê thay vì đoạn văn dài
- Không tham chiếu dự án riêng lẻ — viết cho đối tượng chung của team
- Đặt tên file theo kebab-case, phản ánh chủ đề (ví dụ: `market-data-pipeline.mdx`)
- Cập nhật `docs.json` đồng thời với việc thêm trang để tránh navigation bị lệch

<Tip>
  Cần trợ giúp trong quá trình đóng góp, mở issue trên repo hoặc liên hệ team qua kênh nội bộ `#quant-docs`.
</Tip>
